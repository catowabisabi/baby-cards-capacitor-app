## What to do if a batch crashes

The patched Crashes:

- **SystemError Errno 22 on stderr flush** → ComfyUI patches probably reverted (electron desktop update). Re-apply `trange→range` + KJNodes `print→pass`.
- **CUDA OOM** → kill cached models, restart ComfyUI, retry.
- **`RuntimeError: 搵唔到 ComfyUI`** → server down; check `127.0.0.1:8188` listening (`netstat -ano | findstr 8188`).

Each `--theme <name> --start N` is idempotent — it overwrites files in place.
You can re-run a failed batch with the same args.

## Manual vs auto discovery

`scripts/comfyui_generate.py` and `scripts/comfyui_cards.py` use
`discover_server()` to ping `127.0.0.1:8188` → `127.0.0.1:8000` →
`localhost:8188` → `localhost:8000`. Override via:

- `--server HOST:PORT` (CLI flag)
- `$env:COMFYUI_SERVER = "HOST:PORT"` (env var, takes priority)
